Overview

UvTool is an application to allow a user to do various operations on a UniVerse server. The program is written in Java and utilizes JavaFX and UniObjects for Java technologies. Its a work in progress, so there may be some issues. Having said that, I've been using it for quite some time and most (if not all) of the major bugs have been fixed.

The application has 4 primary functions:

  • Running a java program locally to read/write UniVerse data
  • Copying data from one UniVerse server to another
  • Copying source code from one UniVerse server to another
  • Comparing two versions of source code via a user-specified third party program

The main window


The application window has 4 buttons:
  • Run - Runs a pre-defined application
  • Compare - Utilizes user-defined external difference program to compare source code
  • Pull/Push - Copies source code items between servers
  • Copy - Copies data records from one server to another

Run

NOTE:This feature currently presumes you know how to write Java code and use UniObjects© for Java. The plan is for UvTool to create a program itself by allowing the user to simply define the interface. But that's a long way off.

The allows you to select and run a previously defined java program. See application definition for more information.

Compare

The compare button opens a dialog box allowing you to select a source server and file, then compare to the same file on a destination server.

Pull/Push

Pull/Push will copy source code items from server to server.

Copy

Copy is used primarily to copy data records from one server to another.

In order to use the system you must establish a connection profile. A connection profile is comprised of 3 items: a server, an account and a user name/password. A server can have one or more accounts, and an account may have one or more users.

Server

The basis for all connections is a "Server". To create a server, click File->New->Server from the main menu:

This will present the server setup dialog.

"Server name" is simply a name to call the server. "Host" is the actual network host name or IP address of the UniVerse server you'll be connecting to.

Account

Once a server is defined, you can then define one or more UniVerse accounts that exist on the server. To do this, from the main menu click File->New->Account

which will bring up the following dialog box:

Select the server on which the account exists.   Next, double click the Account name drop-down menu and type in the UniVerse account name.   Finally, enter the full path name of the account. You may need to consult a system administrator to get this info.

Now you can complete the connection by setting up a profile.

Profile

A profile is where you enter your logon credentials to a given account on a given server.   To create a profile, from the main menu, click File->New->Profile
which will present the profile setup dialog box:

            NOTE: At this writing, some of the features of the profile create dialog may not be working smoothly.  
            Particularly, the "+" buttons which are supposed to let you define a new server or account on the fly.
The "Test connection" and "Save" buttons will be disabled until all the required information is entered.   Select the server and accounts from the drop-down box for each.   Next, enter your user name and password in the appropriate boxes.   Next, click the "Test connection" button. If the connection succeeds, the Profile Title drop-down will be cleared. Enter a title for this profile, such as "admin on corvette", then click the Save button.

Usage

  • Application definition

To run a java application, you must first write one, then define the interface. Writing the program is beyond the scope of this document. Refer to Rocket's UniObjects for Java documention for more information.

To define a program, select File->New->Application from the main menu and the following dialog box will be presented

Fill out the name of the application class, the package in which the class resides, and a description of what the application does. The files that the application uses is split into two sections: Files that are read from and files that are written to. The reason for the two sections is to allow an application to read from one server, and write to another.
